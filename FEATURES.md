| US-S01 | As a host, I run `s2c_server -p 1122` and the server becomes reachable to anyone on my network.                                                                                                                                                                                    | `s2c-server` is installed and port is free                     | CLI parses `-p/--port` (default 1122)                                           | Server binds `0.0.0.0:<port>` and starts the accept loop                                            | ✅     | `Server.__init__`, `_setup_socket`; `tests/test_cli.py::test_server_constructor_binds_listen_socket`, `::test_server_main_help_lists_port_flag` | SO_REUSEADDR is set; Phase 5 covered by direct Server(port=0) and `--help` surface tests. |
| US-S02 | As a host, I want clients to keep connecting without manual restarts.                                                                                                                                                                                                              | Server is listening                                            | `socket.accept()` returns                                                      | A daemon thread runs `_handle_client` for each conn                                                  | ✅     | `serve_forever`                     | One thread per conn — fine for ≤5 peers/room. No global cap. |
| US-S03 | As a room owner, peers joining with the same `session_id` should see each other.                                                                                                                                                                                                   | Two clients connect with `{i, s}` matching                     | First packet from each                                                          | Both are mapped to `rooms[session_id][client_id] = {"c": sock}`                                       | 🟡     | `_save_room`                        | `cid` overwrites previous user with same id; max 5 enforced but **dead peers never removed** (memory leak). |
| US-S04 | As a server, I want at most 5 active peers per room to bound resources.                                                                                                                                                                                                            | Room already has 5 peers                                       | 6th peer sends `{i, s}`                                                         | 6th peer is silently dropped                                                                         | ✅     | `_save_room` len check              | Drops are silent because broadcast try-except swallows errors. |
| US-S05 | As a peer, my packets should reach everyone else in my room.                                                                                                                                                                                                                      | Peer A in room `R` sends `{i, s, v\|a}`                       | Server receives via `Framer.feed()`                                            | Bytes are forwarded verbatim (`encode_line(pkt)` once) to every other socket in `R`                   | ✅     | `_route_frame`, `_broadcast`         | NDJSON + raw-byte fan-out fix the framing bug; serialisation done ONCE per frame. |
| US-S06 | As a sysadmin, I want to recover from port-in-use errors.                                                                                                                                                                                                                          | Port 1122 is held by another process                          | `s2c-server -p 1122` is run                                                     | Server prints a warning and retries the bind every 1s until success                                  | ✅     | `_setup_socket` while-loop; `tests/test_cli.py::test_server_port_in_use_raises_with_bind_retry_false` | `bind_retry=False` raises EADDRINUSE (covered); retry loop itself has a 1-s sleep so exhaustive CI test coverage is deferred to Phase 6. |
| US-S07 | As an operator, I want errors logged so I can debug.                                                                                                                                                                                                                              | An exception is raised inside a handler                       | Handler's try/except                                                            | `log.exception(…)` captures the full traceback via the `s2c.server` logger                             | ✅     | `_handle_client`, `_broadcast`, `_handle_runner` | `logging` module wired up at import; tracebacks flow to stderr with timestamps. Verbosity is `-v` / `-vv` on the CLI. |
| US-S08 | **Lack** — As a user, I want the server to know who is in the room right now.                                                                                                                                                                                                     | A peer `{i,s}` joins                                          | Server stores peer                                                              | No `/peers` or telemetry endpoint is exposed                                                         | 🔴     | n/a                                 | No introspection; client retries connection blindly. |
| US-S09 | **Lack** — As a dev, I want a web endpoint that creates a session_id and returns a join command.                                                                                                                                                                                  | n/a                                                            | n/a                                                                             | Server only exposes a raw TCP stream                                                                  | ⚪     | n/a                                 | Targeted by zero-install plan (see PLAN.md §P4). |
